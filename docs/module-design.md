# Module Design

## 页面与模块

### 1. Watchlist 自选模块

职责：

- 管理用户关注的股票/ETF/港股等标的
- 展示基础行情摘要
- 作为提醒规则的标的选择入口

当前已做：

- 列表展示
- 本地新增股票
- 市场类型简单推断

后续可扩展：

- 编辑/删除/置顶
- 分组（A股 / 港股 / ETF）
- 搜索联想
- 实时行情刷新

### 2. Alerts 提醒规则模块

职责：

- 管理提醒规则
- 支持多种提醒条件
- 控制启停状态

当前已做：

- 价格高于/低于
- 本地新增规则
- 启用/停用切换

后续可扩展：

- 涨跌幅提醒
- 连续触发去重
- 时间窗控制
- 到价后二次确认
- 多条件组合规则

### 3. History 提醒历史模块

职责：

- 展示提醒触发记录
- 追踪是否通知成功、是否播音

当前已做：

- 静态历史记录列表
- 展示触发价、目标价、时间、声音状态

后续可扩展：

- 按日期筛选
- 搜索
- 清空/导出
- 点击查看对应规则与标的

### 4. Settings / Service 设置模块

职责：

- 展示后台监控与声音提醒的系统状态
- 提供测试按钮和状态说明

当前已做：

- 监控服务占位状态
- 声音提醒占位状态
- 准备后台监控 / 试播提醒音按钮

后续可扩展：

- 通知权限检查
- 电池优化白名单引导
- 声音文件选择
- 轮询频率配置
- 静默时段设置

## 服务模块

### MonitorService

后续建议职责：

- 周期性拉取行情
- 执行规则匹配
- 写入提醒历史
- 触发通知/声音

### AudioAlertService

后续建议职责：

- 预加载音频资源
- 播放默认提醒音
- 支持不同提醒类型用不同声音

## 数据流建议

```text
MarketDataSource -> QuoteRepository -> RuleEngine ->
HistoryRepository + NotificationService + AudioAlertService
```

## 状态管理建议

当前为了降低起步成本，先不引入状态管理框架。

后续如果页面复杂度上来，可以考虑：

- Riverpod（更适合长期维护）
- Bloc（适合流程明确的大项目）
- ValueNotifier / ChangeNotifier（轻量过渡）
